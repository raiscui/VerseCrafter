# 关键洞察

## [2026-03-22 12:18:55 UTC] 主题: 多卡排查时, 终端里“以为启动了什么命令”不如 `/proc/<pid>/cmdline` 可靠

### 发现来源
- 在 `demo_data/my4` 的 8 号镜头 GPU 排查中, 用户口头贴出的命令看起来是双卡参数.
- 但当前真实运行中的父进程和 Step 6 子进程都显示为单卡参数.

### 核心问题
- 对长链路批处理脚本来说, 人很容易把:
  - 终端里准备执行的命令
  - 聊天里复制出来的命令
  - 当前系统里实际跑着的进程参数
  混成同一件事.
- 一旦这三者不一致, 就会把“命令启动层问题”误诊成“代码逻辑悄悄降级”.

### 为什么重要
- 这种误判会直接把排查方向带偏.
- 人会去怀疑:
  - 多卡代码没有生效
  - 第 8 号镜头逻辑有分支
  - 第二张 GPU 调度失败
- 但真实原因可能只是当前进程根本没带上双卡参数.

### 当前结论
- 对这类问题, 最先该看的不是聊天里的命令文本, 而是:
  - `/proc/<pid>/cmdline`
  - 当前是否真的存在 `torchrun`
  - manifest / 中间产物里记录的最终设置
- 只有这三者一致时, 才值得继续怀疑代码内部的多卡逻辑.

### 后续讨论入口
- 后续若继续改善 `single_image_multi_trajectory.py`, 可以考虑在启动时更显式地打印:
  - 本轮有效的 Step 6 关键参数
  - 是否会使用 `torchrun`
  - 每个镜头最终调用的推理命令摘要

## [2026-03-25 13:12:09 UTC] 主题: 用 `pgrep -af` 做接力等待时, 匹配模式可能把等待脚本自己也算进去

### 发现来源
- 在 `my6` 完成后接手启动 `my7` 的过程中, 自动接力脚本长时间没有放行.
- 复盘后发现真实的 `my6` 进程早已结束, 但等待脚本仍持续打印“`my6` 仍在运行”.

### 核心问题
- 当等待脚本自身命令行里就包含被搜索的模式时, `pgrep -af` 会把脚本自己也匹配出来.
- 这样会让“等待直到目标进程退出”的逻辑变成死循环.

### 为什么重要
- 这种问题不是业务代码出错, 但会直接制造“任务没接上”的假象.
- 如果不先核对真实进程和 manifest, 很容易误判成推理脚本没执行、resume 有 bug, 或者多卡阶段卡死.

### 未来风险
- 以后只要继续用“脚本里 `pgrep` 搜自己要等的模式”这一类写法, 都可能复现同样的问题.
- 特别是在命令参数很长、模式文本被完整内嵌进 shell 脚本时, 风险更高.

### 当前结论
- 这次自动接力失败的根因不是推理流程, 而是等待脚本的进程匹配策略.
- 更稳的等待方式应避免自匹配, 例如:
  - 直接记录目标 PID 并 `wait`
  - 或使用不会把当前脚本自身命中进去的匹配方式

### 后续讨论入口
- 后续若再写类似“等待任务 A 结束再启动任务 B”的编排脚本, 应优先设计为 PID 驱动, 不要只靠模糊文本匹配.

## [2026-03-25 18:31:20 UTC] 主题: 在仓库根目录用系统 Python 查包版本时, 本地源码目录可能伪装成已可导入模块

### 发现来源
- 本次核对 `PyTorch` / `pytorch3d` 真实版本时, 普通 `python3` 在仓库根目录下 `import pytorch3d` 成功, 但并非来自已安装发行版.

### 核心问题
- 工作区存在顶层 `pytorch3d/` 源码目录时, 系统 Python 可能把它当成 namespace package 导入.
- 这会制造一种错觉: “模块能导入, 所以包已经装好了”.
- 但此时 `__file__` 可能是 `None`, `__version__` 也可能不存在, `importlib.metadata` 还会提示没有已安装发行版.

### 为什么重要
- 这会直接污染版本排查.
- 人可能把“源码目录可导入”误判成“当前环境已经正确安装并可运行”.

### 当前结论
- 对这类仓库, 查真实安装版本时应优先看:
  - `pixi run python -m pip show ...`
  - `importlib.metadata.version(...)`
  - 模块 `__file__` 是否位于 site-packages
- 仅凭 `import xxx` 成功, 不能证明包已正确安装.

### 后续讨论入口
- 以后若再核对类似本地源码安装依赖, 先避开仓库根目录或同步检查 `__file__` / `metadata`.
